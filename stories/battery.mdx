## The challenge

Battery's investment team works through the same questions daily: which companies are raising, who's behind them, what does the technical footprint look like. The answers were scattered. Company data in Crunchbase, relationship history in Salesforce, technical signals on GitHub, internal notes somewhere else.

They needed one interface that could pull it all together, normalize it, and stay useful across browsers, tabs, and time zones. Simple on paper. Complicated in practice.

## What we built

The Battery platform is a set of tools pulling from the same data layer.

The **web apps** handle deal research, portfolio analysis, and complex queries across sources. Built for people who live in investment data all day. Density and speed over visual minimalism.

The **Chrome extension** brings it into the browser. An analyst on a company page or GitHub profile pulls up relevant data without switching tabs.

Underneath sits a **backend** for data ingestion, normalization, and query processing. Data from Crunchbase, Salesforce, GitHub, and other sources flows through dedicated pipelines into a shared model.

## The query engine

Investment professionals don't search like most users. They build queries like: "Series B healthcare companies that raised in the last 18 months, where we have a relationship, and engineering headcount grew 30%+."

That hits multiple data sources, needs real time joins, and has to come back fast. The system uses indexed, pre-joined datasets with full text search, faceted filters, and boolean logic.

The user just types and gets results. Making that work required careful data modeling, query optimization, and a UI that handles complex search without getting in the way.
